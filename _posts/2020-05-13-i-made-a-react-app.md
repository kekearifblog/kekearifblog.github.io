I finally got around to toying around with React. I have been wanting to dip
into web development for some time but didn't have a very good foundation in
HTML and CSS. I solved that after stumbling across the beautiful tutorials at
[Interneting is Hard](https://www.internetingishard.com). If you know nothing
and want to learn something, then go there now. Highly recommended.

With my basic HTML and CSS foundation I then went through the official React
documents. They are very well written (unlike Apple's) and it seemed to be
enough because I was able to glue together a quick iOS Calculator Clone app.

I know there are millions of clone iOS Calculator apps out there. What's better
about mine? Nothing. Or maybe the UI is more accurate than other clones. The
point of this app was just to build something manageable that would solidify
what I had just learned about React components and rendering. I resisted the
urge to implement "perfect" calculator logic and focused most of my efforts on
the React parts. I think a lot of developers get side tracked and obsessed with
the wrong things or the newest technology. I'm starting to realize that
obsession management is a skill in itself if you ever want to ship anything.

So far I've really enjoyed my experience with React, being able to split
functionality into components is a good fit for me, it's similar to how I think
when making Swift apps. Even if a React app is overkill for simple pages I
would still probably use it because HTML without components seems so Alien to
me.

Check out the project source code at [https://github.com/kekearif/react-ios-calculator-clone](https://github.com/kekearif/react-ios-calculator-clone). The
